Uitleg over commentaar en documentatie (HTML & CSS)

In dit project is bewust gewerkt met duidelijke structuur en eenvoudige classes zodat de code makkelijk 
te begrijpen is. De HTML-bestanden zijn logisch opgebouwd met vaste onderdelen zoals header, main, 
section en footer. Dit zorgt ervoor dat elke pagina dezelfde opbouw heeft en gemakkelijk aanpasbaar blijft.


In de HTML-code zijn commentaarregels toegevoegd om aan te geven:

- Waar een nieuw onderdeel begint (zoals productoverzicht, footer of admin-gedeelte)

- Wat het doel is van bepaalde blokken (bijvoorbeeld eerste productoverzicht, tweede overzicht, admin-tabel)

- Waarom bepaalde delen apart staan (zoals verschillende productpagina’s)



De CSS is opgesplitst per functie:

- Algemene styling (body, header, navigatie)

- Productcards en labels (Nieuw, Sale, Limited)

- Product detail pagina

- Footer

- Media query voor mobiel


De gebruikte class-namen zijn bewust simpel en duidelijk gekozen (product, products, product-detail, 
footer-container, footer-column) zodat meteen duidelijk is waarvoor ze dienen.
Dit maakt het makkelijker voor een andere programmeur om snel te zien welke CSS bij welk onderdeel hoort.

Er is ook een media query toegevoegd om te tonen dat de website responsiev is. Hierbij is in commentaar 
uitgelegd dat deze bedoeld is voor mobiele schermen.

Door deze aanpak met duidelijke commentaarregels en gestructureerde code is het voor iedereen die
aan het project werkt eenvoudig om de code te begrijpen, aan te passen en uit te breiden indien nodig.
